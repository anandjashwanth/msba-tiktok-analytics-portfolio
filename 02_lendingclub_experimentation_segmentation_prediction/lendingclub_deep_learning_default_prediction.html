1. Feature Selection and Justification

The table below documents every variable selected, its encoding type, and the credit-risk rationale drawn from the LendingClub data dictionary.

Two core rules were applied before selecting any variable:

Rule 1 — No post-origination leakage. Variables that are only observable after a loan has already defaulted or been repaid (e.g. total_pymnt, recoveries, out_prncp) are excluded. Including them would let the model 'see the future' and produce artificially inflated test scores that would not generalise to real lending decisions.

Rule 2 — No near-duplicate features. funded_amnt correlates with loan_amnt at r = 0.9997 and carries no independent signal, so it is dropped. installment is mathematically derived from loan_amnt, int_rate, and term, so it adds no new information once those three are already included.

df['int_rate']   = pd.to_numeric(df['int_rate'], errors='coerce')

# ── Binary encoding: term ─────────────────────────────────────────────────
# Only two values (36 and 60 months) — a single binary column is sufficient
df['term_bin'] = (df['term'] == 60.0).astype(int)  # 0 = 36 months, 1 = 60 months

# ── Ordinal encoding: grade ───────────────────────────────────────────────
# Grade is explicitly ordered A (safest) → G (riskiest), so we assign integers
# that preserve that ordering. The model can then learn monotone relationships.
grade_map = {'A': 1, 'B': 2, 'C': 3, 'D': 4, 'E': 5, 'F': 6, 'G': 7}
df['grade_ord'] = df['grade'].map(grade_map)

# ── One-hot encoding: home_ownership, purpose, verification_status ────────
# These are nominal — there is no natural ordering between categories.

                    drop_first=True, dtype=int)

# Target variable

2c. Undersampling class 0 to balance the dataset

The dataset has a ~5.5:1 class imbalance — roughly 85% good loans and 15% defaults. A model trained naively will learn to predict 'good loan' for almost everything and still achieve high accuracy while catching almost no defaults.

We address this with random undersampling of the majority class (class 0). We randomly draw a sample of class 0 rows equal to the number of class 1 rows, then combine them to produce a perfectly balanced 50/50 dataset. Undersampling is preferred here over oversampling (e.g. SMOTE) because:

  • The dataset is large (~48k rows) so discarding majority rows still leaves ample training data
  • It avoids introducing synthetic minority samples that may not reflect real borrower profiles

Important: Undersampling is applied before the train/val/test split. This means the test set is also balanced, which is intentional for fair metric comparison across segments. In a production setting you would undersample only the training set.

3. Model Definition

We define our model as a class inheriting from nn.Module, exactly as shown in the 7_1_Optimised_DNN notebook. The architecture is a funnel DNN: layers compress from 128 → 64 → 32 neurons, forcing the network to distil increasingly abstract representations before the single output neuron.

Each hidden block follows Linear → BatchNorm1d → ReLU → Dropout:

  • BatchNorm1d normalises activations within each mini-batch, stabilising gradients and allowing faster training
  • ReLU introduces non-linearity ($f(x) = \max(0, x)$) so the model can learn patterns beyond linear combinations
  • Dropout randomly zeros a fraction of neurons each pass, preventing co-adaptation and acting as an ensemble regulariser

The output layer returns a single raw logit — BCEWithLogitsLoss applies sigmoid internally.

Code cell 21
class DefaultDNN(nn.Module):
    def __init__(self, input_dim, dropout_rate=0.3):
        super(DefaultDNN, self).__init__()

        self.network = nn.Sequential(
            # Layer 1
            nn.Linear(input_dim, 128),
            nn.BatchNorm1d(128),
            nn.ReLU(),
            nn.Dropout(dropout_rate),

            # Layer 2
            nn.Linear(128, 64),
            nn.BatchNorm1d(64),
            nn.ReLU(),
            nn.Dropout(dropout_rate),

            # Layer 3
            nn.Linear(64, 32),
            nn.BatchNorm1d(32),
            nn.ReLU(),
            nn.Dropout(dropout_rate),

            # Output Layer — single logit
            nn.Linear(32, 1)
        )

    def forward(self, x):
        return self.network(x)

4. Hyperparameter Tuning

We use a sequential one-factor-at-a-time search: each round varies one hyperparameter while holding all others at their current best value. Validation AUC-ROC is used to select the winner each round.

Memory is freed after every run (del model, torch.cuda.empty_cache(), gc.collect()) to prevent back-to-back runs exhausting RAM.

Code cell 23
def train_and_validate(input_dim, hidden_layers, lr, weight_decay,
                        batch_size, dropout_rate, patience, epochs=100):
    """
    Builds and trains one model configuration.
    Returns (val_auc, best_val_loss). Cleans up memory before returning.
    """

    # Build model dynamically from hidden_layers list
    layers = []
    in_features = input_dim
    for out_features in hidden_layers:
        layers += [
            nn.Linear(in_features, out_features),
            nn.BatchNorm1d(out_features),
            nn.ReLU(),
            nn.Dropout(dropout_rate)
        ]
        in_features = out_features
    layers.append(nn.Linear(in_features, 1))
    model = nn.Sequential(*layers).to(device)

    criterion = nn.BCEWithLogitsLoss(pos_weight=pos_weight)

    # AdamW adds weight decay directly to the weight update rule for better regularization
    optimizer = optim.AdamW(model.parameters(), lr=lr, weight_decay=weight_decay)

    # Learning Rate Decay: Reduces LR when validation loss plateaus
    scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, mode='min', factor=0.5, patience=5)

    t_loader = DataLoader(TensorDataset(X_train_t, y_train_t), batch_size=batch_size, shuffle=True)
    v_loader = DataLoader(TensorDataset(X_val_t,   y_val_t),   batch_size=batch_size, shuffle=False)

    best_val_loss     = float('inf')
    epochs_no_improve = 0
    early_stop        = False
    best_state        = None

    for epoch in range(epochs):
        if early_stop:
            break

        # --- Training Phase ---
        model.train()
        running_loss = 0.0
        for inputs, labels in t_loader:
            inputs, labels = inputs.to(device), labels.to(device)

            optimizer.zero_grad()
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()

            running_loss += loss.item() * inputs.size(0)

        # --- Validation Phase ---
        model.eval()
        val_loss = 0.0
        with torch.no_grad():
            for inputs, labels in v_loader:
                inputs, labels = inputs.to(device), labels.to(device)
                outputs = model(inputs)
                loss = criterion(outputs, labels)
                val_loss += loss.item() * inputs.size(0)

        epoch_val_loss = val_loss / len(v_loader.dataset)

        # Update Learning Rate Scheduler based on validation loss
        scheduler.step(epoch_val_loss)

        # --- Early Stopping Logic ---
        if epoch_val_loss < best_val_loss:
            best_val_loss     = epoch_val_loss
            epochs_no_improve = 0
            best_state        = {k: v.clone() for k, v in model.state_dict().items()}
        else:
            epochs_no_improve += 1
            if epochs_no_improve >= patience:
                early_stop = True

    # Load the best model weights found during training
    model.load_state_dict(best_state)
    model.eval()

    val_probs, val_labels_list = [], []
    with torch.no_grad():
        for inputs, labels in v_loader:
            probs = torch.sigmoid(model(inputs.to(device))).cpu().numpy().flatten()
            val_probs.extend(probs)
            val_labels_list.extend(labels.numpy().flatten())

    val_auc = roc_auc_score(val_labels_list, val_probs)

    # ── Free memory before returning ──────────────────────────────────────
    # Without this, each tuning run leaves its model and optimiser on the
    # GPU/CPU heap and back-to-back runs will exhaust available RAM.
    del model, optimizer, scheduler, criterion
    del t_loader, v_loader, best_state
    if torch.cuda.is_available():
        torch.cuda.empty_cache()
    gc.collect()

    return val_auc, best_val_loss

epochs           = 100
patience         = 20
best_val_loss    = float('inf')
epochs_no_improve = 0
early_stop       = False

train_losses, val_losses = [], []

for epoch in range(epochs):
    if early_stop:
        print(f'Early stopping triggered at epoch {epoch}')
        break

    # --- Training Phase ---
    model.train()
    running_loss = 0.0
    for inputs, labels in train_loader:
        inputs, labels = inputs.to(device), labels.to(device)

        optimizer.zero_grad()
        outputs = model(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        running_loss += loss.item() * inputs.size(0)

    epoch_train_loss = running_loss / len(train_loader.dataset)
    train_losses.append(epoch_train_loss)

    # --- Validation Phase ---
    model.eval()
    val_loss = 0.0
    with torch.no_grad():
        for inputs, labels in val_loader:
            inputs, labels = inputs.to(device), labels.to(device)
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            val_loss += loss.item() * inputs.size(0)

    epoch_val_loss = val_loss / len(val_loader.dataset)
    val_losses.append(epoch_val_loss)

    # Update Learning Rate Scheduler based on validation loss
    scheduler.step(epoch_val_loss)

    # --- Early Stopping Logic ---
    if epoch_val_loss < best_val_loss:
        best_val_loss     = epoch_val_loss
        epochs_no_improve = 0
        # Save the best model
        torch.save(model.state_dict(), 'best_model.pth')
    else:
        epochs_no_improve += 1
        if epochs_no_improve >= patience:
            early_stop = True

    if (epoch + 1) % 5 == 0:
        print(f'Epoch {epoch+1:03d}/{epochs} | Train Loss: {epoch_train_loss:.4f} | Val Loss: {epoch_val_loss:.4f}')

cluster_cols = ['loan_amnt', 'int_rate', 'annual_inc_log', 'dti',
                'open_acc', 'revol_bal', 'revol_util', 'total_acc']

df_c['annual_inc_log'] = np.log1p(df_c['annual_inc'])

# Mahalanobis outlier removal (mirrors Q2)
X_c     = df_c[cluster_cols].values
cov_inv = np.linalg.inv(np.cov(X_c.T))
diff    = X_c - X_c.mean(axis=0)
m_dist  = np.sqrt(np.einsum('ij,jk,ik->i', diff, cov_inv, diff))
df_c    = df_c[m_dist <= np.percentile(m_dist, 99.9)].reset_index(drop=True)

del X_cs
