## Project 1

Retail Inventory SQL Analytics for PopBeauty

**Title**
SQL Business Analyst — E-commerce Inventory and Fulfilment Optimisation

**Project URL**
[GitHub project link]

**Description**

## Project 2

**Project name**
Loan Policy A/B Testing and Borrower Segmentation

**Title**
Experimentation and Customer Segmentation Analyst

**Project URL**
[GitHub project link]

**Description**
Built an analytical workflow using LendingClub data to compare two rule-based loan approval policies through a simulated A/B testing framework. The project defined a primary business metric, average net return per application, and supporting diagnostic metrics including false positive rate, false negative rate and approval rate. I then extended the analysis with borrower segmentation using clustering methods, creating four borrower profiles and comparing policy performance across each segment. The project shows how aggregate metrics can hide segment-level risks and opportunities, and how data-driven segmentation can improve strategy design. This is relevant to TikTok Shop and TikTok LIVE because it demonstrates experiment design, KPI thinking, user/customer segmentation, behavioural analysis, and translating quantitative results into strategic recommendations.

## Project 3

Deep Learning Default Prediction with Segment-Level Evaluation

**Title**
Python Data Science Modelling and Risk Prediction Project

**Project URL**
[GitHub project link]

**Description**
Developed a PyTorch dense neural network to predict loan default using LendingClub application data. The workflow included feature selection, leakage prevention, categorical encoding, missing value treatment, class balancing through undersampling, train-validation-test splitting, scaling, neural network design, hyperparameter tuning, early stopping, test evaluation and segment-level performance analysis. ## Project 4

**Project name**
Revenue and Capacity Optimisation for Rental Operations

**Title**
Operations Strategy and Revenue Optimisation Analyst

**Project URL**
[GitHub project link]

**Description**
Built a linear optimisation model in Python using Pyomo to improve rental acceptance decisions for a generator fleet across a 52-week planning horizon. The model used demand, price, rental duration, fleet size, returns and inventory availability to decide how many rental requests to accept while respecting capacity constraints. The analysis compared actual and optimised performance, showing higher total revenue, improved load factor and stronger return on investment. It also used shadow prices to identify weeks where additional capacity would have the highest marginal value. This project is relevant to strategy and business analytics roles because it demonstrates planning, operational decision-making, constraint-based optimisation, commercial metric interpretation and practical recommendation development.
